## 三、登录流程测试用例

### 场景1：家属首次登录（未绑定）
```
1. 访问登录页面
2. 选择"家属端"
3. 完成微信授权登录
4. 系统检查 family_members 表
5. 发现无绑定记录
6. 自动跳转到 bind-senior 页面
7. 输入长者姓名和验证码
8. 绑定成功后跳转到 home 页面
```

### 场景2：家属再次登录（已绑定）
```
1. 访问登录页面
2. 选择"家属端"
3. 完成微信授权登录
4. 系统检查 family_members 表
5. 发现已有绑定记录
6. 自动跳转到 home 页面
7. 显示长者信息
```

### 场景3：员工登录（有权限）
```
1. 访问登录页面
2. 选择"管理端"
3. 完成微信授权登录
4. 系统检查 employee 表
5. 发现记录，验证通过
6. 自动跳转到 admin-home 页面
```

### 场景4：非员工尝试登录管理端
```
1. 访问登录页面
2. 选择"管理端"
3. 完成微信授权登录
4. 系统检查 employee 表
5. 未发现记录
6. 显示"权限不足"提示
7. 停留在登录页面
```

## 五、快速测试指南

### 测试家属登录流程
```bash
# 1. 已绑定家属（王建国）
# 登录后应直接跳转到 home 页面
familyId: family_001

# 2. 未绑定家属（赵敏）
# 登录后应跳转到 bind-senior 页面
familyId: family_007

# 3. 多长者绑定家属（王丽华）
# 登录后跳转到 home 页面，需支持多长者切换
familyId: family_005
```

### 测试员工登录流程
```bash
# 1. 护理员登录
openid: wx_nurse_001
# 应成功登录并跳转到 admin-home

# 2. 管理员登录
openid: wx_admin_001
# 应成功登录并拥有全部权限

# 3. 普通员工登录
openid: wx_staff_001
# 应成功登录但权限受限

# 4. 非员工尝试登录
# 使用任意未注册 openid
# 应提示"权限不足"
```
